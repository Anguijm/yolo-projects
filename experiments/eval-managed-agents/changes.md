# Changes — eval-managed-agents implementation

Infrastructure tick (autonomous cron, drain mode). Creates only files under
`experiments/eval-managed-agents/` — no root project dir, no `index.html`, no
changes to tick-tock dispatch code (measure-and-recommend only).

## Files created (deliverable_paths)

### `benchmark.py` (~290 lines)
A two-mode benchmark harness comparing Managed Agents vs manual tick-tock orchestration.

- `CANONICAL_TASK` (dict) — a fixed multi-step dev-task spec (one YOLO build as an
  ordered work-list). Deterministic; no live repo state.
- `DIMENSIONS` (list) — 7 comparison dimensions (`agent_loop`, `context_passing`,
  `tool_execution`, `error_recovery`, `determinism_control`, `observability_audit`,
  `cost_latency`), each with what manual owns, what managed owns, the lower-code
  winner, and a rationale grounded in documented API behavior.
- `run_static() -> dict` — pure/offline scorer. Tallies winners, computes the
  `orchestration_ownership_delta` (sheddable-if-managed vs retained-under-manual),
  returns a structured result + an `iterate` verdict and recommendation.
- `run_live(task, model, deadline_s=240) -> dict` — gated live benchmark. Lazily
  imports `anthropic`; creates the agent ONCE, an environment, a session; streams
  events stream-first; accumulates tokens from `span.model_request_end.model_usage`;
  breaks on terminal `session.status_idle` (stop_reason != requires_action) or
  `session.status_terminated`; bounded by a wall-clock deadline so it can't hang.
  Raises a friendly `SystemExit` if `ANTHROPIC_API_KEY` or the SDK is missing.
- `write_results_md(result, path)` + `_dimension_table(result)` — render the
  committed `results.md` from a static run (regenerated, so it never drifts).
- `main(argv=None)` — argparse CLI: default static (writes results.md), `--json`
  machine summary, `--live` opt-in, `--out` override.

SDK surface used in `run_live` is the documented managed-agents-2026-04-01 path:
`client.beta.agents.create` / `environments.create` / `sessions.create` /
`sessions.events.stream` / `.send`. Agent is created once (not per request-path).

### `results.md`
Generated by `python3 benchmark.py` (static). Premise, the 7-dimension table,
score (managed-wins=4, manual-wins=3, sheddable=4/7), ownership delta, and the
**ITERATE/DEFER** verdict. Reproducible — no live numbers, no embedded timestamps
in the scored body.

### `README.md`
Explains the premise (verified against the first-party API), how to run static +
live, the gating rationale (managed sessions run minutes / bill tokens → never
auto-run from cron), and the deferred-adoption verdict.

## Council (advisory — never blocks the drain)
Four gates ran; objections were cheaply and genuinely fixed rather than waved through:
- **PLAN** — 7/7 APPROVE.
- **IMPLEMENTATION** — BUGS(high, live resource leak) + SECURITY(medium, path traversal on `--out`) + UI(no live progress). Fixed: added `finally`-block cleanup of session/env/agent + live progress logging.
- **TESTS** — BUGS now APPROVE; SECURITY(high) flagged `unrestricted` env networking. Fixed: switched the live env to deny-by-default `limited` networking (benchmark task needs no egress).
- **OUTCOME** — SECURITY APPROVE; BUGS(critical) flagged cleanup unreliability on partial create. Fixed: resources init to `None`, created inside the `try`, cleanup guards each handle so a partial-create failure can't orphan resources or `NameError`.

The remaining advisory note (path traversal on operator-controlled `--out`) is low-risk for an internal CLI and left as-is. Verdicts logged in `council_{plan,implementation,tests,outcome}.json`.
