# Plan ŌĆö eval-managed-agents (infrastructure tick)

## Goal
Benchmark Claude Managed Agents (first-party `/v1/agents` + `/v1/sessions` orchestration) against the repo's current manual tick-tock orchestration on a representative multi-step dev task, and decide whether managed agents can simplify tick-tock dispatch logic.

## Scope
**In scope** ŌĆö a self-contained, offline-runnable benchmark harness (`benchmark.py`) that:
- Encodes a canonical multi-step dev-task spec (one historical YOLO build, replayed as a fixed work-list).
- Models both orchestration paths and scores them across fixed dimensions (orchestration LOC you maintain, round-trip/loop ownership, statefulness/context-passing, error-recovery ownership, tool-execution hosting).
- Carries a **correct** live-benchmark path (create agent once ŌåÆ create session ŌåÆ stream events ŌåÆ measure wall-clock + tokens) gated behind `--live` + `ANTHROPIC_API_KEY`, so the harness is real, not a mock.
- Emits a machine-readable JSON summary and a human-readable `results.md`.

**Explicitly NOT in scope:**
- Running a live Managed Agents session inside this cron (sessions run minutes; violates the synchronous-finish rule and burns cost). The live path exists but is never auto-invoked here; CI runs the static path only.
- Changing any tick-tock dispatch code. This tick only *measures and recommends*; adoption is a separate future tick.
- Creating a root-level `<name>/` project dir or any `index.html`. Infrastructure ticks live only under `experiments/<name>/`.
- Migrating `google-generativeai` ŌåÆ `google-genai` (tracked separately in CLAUDE.md; out of scope here).

## Approach
Subtasks, in sequence:
1. **Task spec (`CANONICAL_TASK`)** ŌĆö a fixed dict describing a multi-step build (plan ŌåÆ implement ŌåÆ 4 council gates ŌåÆ test ŌåÆ ship) with an ordered step-list. No dependency on live state; deterministic.
2. **Dimension model (`DIMENSIONS`)** ŌĆö list of comparison dimensions, each with a `manual` and `managed` characterization and a `winner` + one-line rationale grounded in documented API behavior (stateful sessions, server-run loop, built-in compaction/prompt-caching/rescheduling, per-session container).
3. **Static scorer (`run_static`)** ŌĆö tallies dimension winners, computes an orchestration-ownership delta (LOC/loop/retry surface the repo would *stop* maintaining), and returns a structured result + verdict.
4. **Live path (`run_live`)** ŌĆö gated; uses the documented SDK surface: `agents.create` (once), `environments.create`, `sessions.create`, `sessions.events.stream`/`.send`, correct idle/terminated break gate; accumulates `span.model_request_end.model_usage` tokens and wall-clock. Returns metrics; never called without `--live`.
5. **Emitters** ŌĆö `write_results_md` (human) + JSON to stdout/`results.json`. `results.md` is regenerated from the static run so it never drifts.
6. **CLI** ŌĆö `argparse`: default static; `--live` opt-in; `--json` to print machine summary; `--out` for results path.

Sequencing: 1ŌåÆ2 feed 3; 3 produces the artifact the OUTCOME gate verifies; 4 is independent and dormant in CI; 5/6 wrap.

## Function Map
`benchmark.py`:
- `run_static() -> dict` ŌĆö score dimensions, compute ownership delta, build result dict + verdict.
- `run_live(task: dict, model: str) -> dict` ŌĆö live wall-clock/token benchmark via Managed Agents SDK; gated.
- `write_results_md(result: dict, path: str) -> None` ŌĆö render `results.md` from a static result.
- `_dimension_table(result: dict) -> str` ŌĆö markdown table helper.
- `main(argv=None) -> int` ŌĆö argparse CLI; default static, `--live`/`--json`/`--out`.
Module-level data: `CANONICAL_TASK: dict`, `DIMENSIONS: list[dict]`.

## Security
No secrets written to disk. The live path reads `ANTHROPIC_API_KEY` from env only (never logged, never echoed into `results.md`/JSON). Static path makes zero network calls. No untrusted input executed. Threat boundary: the harness is read-only w.r.t. repo state except the three files it writes under its own dir.

## UI
N/A ŌĆö CLI tool. Output states: static result table (default), `--json` machine summary, `--live` adds a measured-metrics block. Empty/error: missing SDK or `ANTHROPIC_API_KEY` under `--live` prints a clear message and exits non-zero without a traceback dump; static mode has no external dependency and always succeeds.

## Guide
README documents: the premise (managed agents = first-party server-run loop + hosted container), exact run commands, how to read the verdict, and the explicit caveat that adoption is deferred pending a live run with a real key.

## Edge Cases
- `--live` without `ANTHROPIC_API_KEY` ŌåÆ friendly error, exit 1.
- `--live` without the `anthropic` SDK installed ŌåÆ friendly error, exit 1.
- Static mode is fully deterministic ŌåÆ `results.md` is reproducible (no timestamps embedded in the scored body; any run-date noted as "generated by static run").
- Live session that never reaches terminal idle ŌåÆ bounded by a wall-clock deadline so it can't hang.

- Run `--json` ŌĆö must emit valid JSON (pipe through `python3 -m json.tool`).
- Confirm no network call in static mode (no `anthropic` import at module top; imported lazily inside `run_live`).
