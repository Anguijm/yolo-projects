## What `timing_extract.py` does (function-level)
- `parse_iso` — tolerant ISO parse.
- `_sanitize(s, maxlen)` — injection-safe markdown sanitizer (addresses SECURITY
  PLAN objection: all strings embedded in REPORT.md are sanitized — newlines
  stripped, `|`/backtick escaped, length-bounded).
- `collect_council_files` — scan `<root>/*/council_<gate>.json` under repo root
  and `experiments/`.
- `extract_local_timings` — per-project gate timestamps + inter-gate deltas;
  marks a build `coherent` only when all gates fall within `--max-session-gap-min`
  AND are monotonic (gate files are overwritten across builds, so a dir can hold
  gate files from different dates — observed 61-day spurious deltas).
- `aggregate_local` — median/max/n per interval over coherent builds; objection
  tally by gate.
- gh path: `gh_available`, `run_cmd` (fixed argv, no shell), `list_recent_runs`,
  `extract_gh_timings` (uses `gh run view --json jobs` — structured step names,
  no YAML parsing; addresses BUGS PLAN objection re: YAML-reformat robustness),
  `aggregate_gh`. Degrades to empty without a token.
- `detect_uncached_installs` — text/regex scan of `tick_tock.yml` for install
  steps with no caching (tolerant of a missing file).
- `identify_bottlenecks` — ranks findings, separating `non_ai` (mechanical) from
  `ai` (inter-gate) stages.
- `render_report`, `build_log`, `selftest`, `main`.

## Key finding (real, from local data)
Median `plan->implementation` interval = 263s over n=10 coherent builds; the one
concrete NON-AI bottleneck: install steps (`pip install ...`, `npm install -g
claude-code`) run UNCACHED on every hourly invocation — fixed overhead removable
with `actions/cache`, independent of any model-speed change.
